# World Capitals Climate Dashboard

## Project Overview

An end to end ETL pipeline that fetches live geographic and weather data
for 10 world capitals, transforms and validates it, loads it into a
SQLite database, and displays it in an interactive Dash dashboard.

## Dashboard Features

### 1. Database Connectivity
The dashboard connects directly to the SQLite database using SQLAlchemy.
It pulls live data from the dim_city and fact_forecast tables using a JOIN query.
If the database is unavailable it falls back to the CSV file automatically.

### 2. KPI Cards
Five summary metrics are displayed at the top of the dashboard:
- Hottest city today with temperature
- Coldest city today with temperature
- Average rain probability across all cities
- Number of cities where travel is recommended
- Number of cities with a climate risk alert

### 3. Visualizations
- World map with color coded city markers by region
- Temperature bar chart comparing high and low temperatures
- Rain probability horizontal bar chart with color coding
- 7 day forecast table with color coded travel recommendations

### 4. Interactivity
- Region dropdown filters all charts simultaneously
- Clicking any city on the map updates the summary card and forecast table
- All charts update dynamically without restarting the app

## Business Insights

The dashboard is designed to answer these real world questions:

**1. Where is the best place to travel this week?**
The travel recommendation field combines temperature and rain probability
to give a simple answer: Highly recommended, Recommended, Carry an umbrella,
or Not recommended. Users can see at a glance which cities are good to visit.

**2. Which cities have extreme weather today?**
The climate risk flag alerts users to extreme heat (above 38C) or heavy rain
(above 80% probability). The KPI card shows how many cities are flagged at a glance.

**3. How does temperature vary across regions?**
The temperature bar chart ranks all 10 cities by today's high temperature,
making it easy to compare cities across Asia, Europe, Americas, Africa, and Oceania.

**4. Which cities are rainy this week?**
The rain probability chart uses color coding: green for low risk, orange for moderate,
and red for high chance of rain. Users can instantly see which cities to avoid
if they want dry weather.

**5. How does location affect temperature?**
Cities closer to the equator like Riyadh, New Delhi, and Jakarta consistently
show higher temperatures than cities at higher latitudes like London and Canberra.

## Data Freshness

Every time etl_complete.py runs, the pipeline:
1. Fetches the latest 7 day forecast from the APIs
2. Deletes all old forecast records from the database
3. Inserts the fresh forecast data
4. Saves the updated CSV

This means the dashboard always shows current data. Run the pipeline once
per day to keep the forecasts up to date.

## Database Schema

### dim_city (dimension table)
Stores one row per capital city. Never deleted, only updated if cities change.

| Column | Type | Description |
|---|---|---|
| city_id | INTEGER | Primary key |
| city | TEXT | Capital city name |
| region | TEXT | World region |
| languages | TEXT | Official languages |
| timezone | TEXT | IANA timezone |
| latitude | REAL | Geographic latitude |
| longitude | REAL | Geographic longitude |

### fact_forecast (fact table)
Stores one row per city per forecast day. Refreshed completely on every pipeline run.

| Column | Type | Description |
|---|---|---|
| forecast_id | INTEGER | Primary key |
| city_id | INTEGER | Foreign key to dim_city |
| date | TEXT | Forecast date |
| high_c | REAL | High temperature in Celsius |
| low_c | REAL | Low temperature in Celsius |
| feels_like_c | REAL | Feels like temperature |
| rain_probability | REAL | Rain chance percentage |
| weather_condition | TEXT | Weather description |
| day_length | TEXT | Hours of daylight |
| season | TEXT | Current season |
| climate_risk | TEXT | Risk flag |
| travel_recommendation | TEXT | Travel advice |

## Switching to PostgreSQL

To switch from SQLite to PostgreSQL change one line in etl_complete.py and dashboard.py:

```python
# Current SQLite
DATABASE_URL = "sqlite:///climate_dashboard.db"

# Switch to PostgreSQL
DATABASE_URL = "postgresql://user:password@host:port/dbname"
```

No other code changes are needed. SQLAlchemy handles everything else automatically.

## Validation Checks

The pipeline runs 5 data quality checks before and after loading:

| Check | What It Validates |
|---|---|
| Null value check | Required fields have no missing values |
| Duplicate detection | No duplicate city and date combinations |
| Temperature range check | All temperatures between -60 and 60 degrees Celsius |
| Row count verification | Exactly 70 rows across 10 cities |
| Referential integrity | All city_ids in fact_forecast exist in dim_city |
